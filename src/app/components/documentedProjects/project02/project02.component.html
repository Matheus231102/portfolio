O componente define vários controladores de formulário (FormControl), cada um com suas próprias regras de validação, e os adiciona a um grupo de formulário principal (FormGroup). As regras de validação incluem:

  • nameInput: Obrigatório, comprimento mínimo de 2 caracteres.
  • usernameInput: Obrigatório, comprimento mínimo de 4 caracteres, sem espaços.
  • emailInput: Obrigatório, formato de e-mail válido.
  • passwordInput e password2Input: Obrigatório, utiliza Regex para validar padrão específico (mínimo de 8 caracteres, pelo menos um dígito, uma letra maiúscula e uma minúscula).

Verificação de Disponibilidade de E-mail e Nome de Usuário

O componente define vários controladores de formulário (FormControl), cada um com suas próprias regras de validação, e os adiciona a um grupo de formulário principal (FormGroup). As regras de validação incluem:

  • E-mail: Após 350ms de inatividade, se o e-mail for válido, uma verificação de disponibilidade é feita.
  • Nome de Usuário: Após 600ms de inatividade, se o nome de usuário for válido, uma verificação de disponibilidade é feita.

Funções de Validação

Algumas funções de validação verificam se os campos atendem aos critérios definidos:

  • Validade do Formulário: Verifica se o formulário como um todo é válido.
  • Disponibilidade das Credenciais: Verifica se tanto o e-mail quanto o nome de usuário estão disponíveis.
  • Validade dos Campos: Funções específicas para verificar a validade de cada campo individualmente.
  • Correspondência de Senhas: Verifica se as duas senhas inseridas são iguais.
  • Critérios de Senha: Funções para verificar se a senha atende aos critérios específicos, como conter pelo menos um dígito, uma letra maiúscula, uma letra minúscula, comprimento mínimo, e uso de caracteres especiais permitidos.

Registro do Usuário

Quando o usuário tenta se registrar, a função `registerUser` verifica se o formulário é válido, se as senhas coincidem e se o e-mail e o nome de usuário estão disponíveis. Se todas as condições forem satisfeitas, a função chama o serviço de registro (`registerService`) para registrar o usuário no back-end. Se o registro for bem-sucedido, um token JWT é recebido e armazenado localmente, e o usuário é redirecionado para a página inicial.
